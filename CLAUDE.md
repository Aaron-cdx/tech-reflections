# CLAUDE.md — tech-reflections

## 项目概述

个人技术沉思录写作项目。每篇文章是对某个技术趋势的深度思考，不是教程或市场报告，而是有观点的个人梳理。

## 写作原则

### 中英文空格规则（核心原则）

**中文与英文字母之间不加空格。** 这是中文互联网原生写作者的自然习惯——加了空格反而「AI味」十足。

- ❌ `AI 编程范式` → ✅ `AI编程范式`
- ❌ `Claude Code 的插件` → ✅ `Claude Code的插件`
- ❌ `让 AI 知道` → ✅ `让AI知道`

**章节编号（数字）与中文标题之间保留空格**，便于阅读：

- ❌ `### 6.2全栈转型的前提` → ✅ `### 6.2 全栈转型的前提`
- ❌ `### 8.1几乎可以确定的` → ✅ `### 8.1 几乎可以确定的`

**数字与中文量词之间不加空格**（正文中的数值表达）：

- ✅ `500万周活`、`2026年`、`3倍`、`第1层`

两个英文单词之间的空格保留（`Claude Code` 不是 `ClaudeCode`），代码块内的内容不受此规则影响。

### 其他写作约定

- **不使用破折号「——」前后的空格**：`思考——这是` 而非 `思考 —— 这是`
- **中文使用全角标点**：`。，！？：；（）【】《》` 而非 `. , ! ? : ; ( ) [ ] < >`
- **代码/文件名/命令使用反引号**：`` `CLAUDE.md` ``、`` `src/payment/**` ``
- **Mermaid 图表语言使用 `mermaid`**（不用 `mermaid` 以外的图表工具）
- **数字与单位之间不加空格**：`500万` 而非 `500 万`

### 文章结构偏好

- 用 `##` 作为一级章节标题（`#` 留给文章主标题）
- 章节编号使用中文数字：`一、` `二、` `三、`
- 子章节使用 `### N.M` 格式
- 每个大章节之间用 `---` 分隔
- 适度使用 Mermaid 图表辅助说明架构关系
- 文末附一段写作后记（`*写作本身就是思考...*`）

### 语气

- 第一人称，个人视角
- 诚实承认不确定的地方（「坦率地说」「这不是预言」）
- 不说「你应该」，说「我的体感是」「可以做的」
- 有判断力但不傲慢
